/////////////////////////////////////////////
///////////// COMANDOS GIT /////////////////
///////////////////////////////////////////

1) CONFIGURAÇÃO INICIAL DO REPOSITÓRIO

git init
- Inicia um novo repositório Git local.

git remote add <nome-remoto> <url-do-repositório>
- Adiciona um repositório remoto ao projeto local.

2) ANTES DE COMMITAR

git status
- Verifica o estado atual dos arquivos no repositório.

git diff
- Mostra as diferenças ainda não commitadas.

git add .
- Adiciona todos os arquivos alterados para a área de stage.

3) SALVAR ALTERAÇÕES

git commit -m "Descrição do commit"
- Cria um commit com as alterações adicionadas.

git push <nome-remoto> main
- Envia os commits para a branch main do repositório remoto.

4) CONSULTAR HISTÓRICO

git log
- Exibe o histórico de commits.

git log --oneline
- Exibe o histórico resumido, com um commit por linha.

git log -p
- Exibe o histórico com detalhes das alterações de cada commit.

5) AJUSTAR O ÚLTIMO COMMIT

git commit --amend -m "Novo texto do commit"
- Altera a mensagem do último commit.
- Também pode ser usado para incluir alterações esquecidas no último commit.

6) DESFAZER ALTERAÇÕES / COMMITS

git revert <id-do-commit>
- Cria um novo commit desfazendo as alterações de um commit específico.
- Indicado quando o commit já foi enviado ao remoto.

git revert --no-commit <commit-inicial>^..<commit-final>
- Reverte uma sequência de commits sem criar o commit final automaticamente.
- Depois, finalize com git commit -m "Revertendo commits".

git reset --soft <id-do-commit>
- Move o ponteiro da branch para um commit anterior, mas mantém as alterações na área de stage.

git reset --hard <id-do-commit>
- Move o ponteiro da branch para um commit anterior e descarta as alterações posteriores.
- Use com cuidado, pois pode apagar trabalho não salvo.
